# Context: Sesion 2026-03-09 - Investigacion de UI

Date: 2026-03-09
Status: ACTIVE
Source Priority: USER_INPUT > CODEBASE > PROJECT_CONTEXT

## Resumen
La sesion se enfoca en documentar investigacion para decidir interfaz grafica de HandsomeTranscribe.
Se comparan rutas desktop Python vs web local con foco en microfono, visualizacion en vivo y consumo de artefactos finales.
Se valida en codigo que el estado actual es pipeline CLI batch sin capa de eventos en tiempo real.
Se mantiene restriccion de costo bajo y local-only por defecto.

## Input del Usuario (fuente principal)
- Necesidad de UI, aun sin decision cerrada entre desktop y web.
- Requisito critico de microfono y permisos.
- Mostrar interlocutores y progreso de meeting en vivo.
- Al final: reproducir audio, transcript y reportes desde la UI.

## Componentes Implicados
- `main.py`

## Flujo Actual
1. Captura WAV.
2. Transcripcion JSON.
3. Diarizacion speaker labels.
4. Resumen.
5. Reportes md/json/pdf.

## Dependencias
- Internas: modulos del pipeline actual.
- Externas: sounddevice, whisper, pyannote.audio, transformers opcional, reportlab, ffmpeg.

## Reglas y Restricciones
- Preservar compatibilidad CLI existente.
- Mantener modo gratuito/local por defecto.
- Introducir UI de forma incremental y testeable por etapa.

## Riesgos de Regresion
- Intentar tiempo real sin arquitectura de eventos puede romper estabilidad.
- Duplicacion de logica entre CLI y UI si no se extrae capa de aplicacion.

## Validacion Recomendada
- Ejecutar tests del pipeline.
- Validar manualmente permisos de microfono y flujo completo fin-a-fin.

## Preguntas Abiertas
- Tecnologia UI inicial (desktop vs web local).
- Nivel exacto de latencia objetivo para vista en vivo.
- Alcance real de soporte multi-dispositivo de audio.
